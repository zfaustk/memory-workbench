# Contributing

This repo is early-stage. Contributions should strengthen proof, clarity, or repeatability.

## Good Early Contributions

- sharper continuity examples
- clearer benchmark rubrics
- schema improvements backed by a concrete failure mode
- small tooling that validates continuity artifacts

## Contributions To Avoid

- generic prompt packs
- broad architecture rewrites with no example
- cosmetic churn without proof improvements

## Working Rule

If a proposed change does not make the repo more `resume-ready`, `handoff-ready`, or `audit-ready`, it is probably not the right first contribution.
